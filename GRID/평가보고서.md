## 2. 수학적 전제 — 로또의 본질

### 2-1. 고정 확률

한국 로또는 1~45에서 6개를 무작위로 추출합니다.

```
1등 확률 = C(45,6)⁻¹ = 1 / 8,145,060 ≈ 0.0000123%
```

이 확률은 다음과 어떤 조건에서도 불변입니다:

- 어떤 번호가 최근에 많이 나왔거나 전혀 나오지 않았어도
- AI가 어떤 점수를 부여했어도
- 직전 회차 번호와 인접해 있어도
- 어느 구간에서 선택했어도

**이전 회차 결과는 다음 회차 추첨에 수학적으로 아무런 영향을 미치지 않습니다.**

## 3. 각 알고리즘 레이어별 평가

### Layer 2-A: 기간별 빈도 분석 (recent_weight: 40)

```python
RECENT_WINDOWS = [(5, 0.20), (10, 0.25), (20, 0.25), (50, 0.18), (100, 0.12)]
```

**설계 의도**: 최근 자주 나온 번호는 앞으로도 나올 가능성이 높다.

**수학적 판정**: ❌ **도박사의 오류 (Gambler's Fallacy)**

> 독립적인 무작위 시행에서 이전 결과의 빈도는 다음 시행 확률에 영향을 주지 않습니다.
> 번호 7이 최근 20회 동안 8번 나왔다고 해서 다음 회차에 더 잘 나오지 않습니다.
> 반대로, 오래 안 나온 번호가 곧 나온다는 것도 동일한 오류입니다.

**실증 검증**: 가중치 40은 전체 점수의 핵심이지만 예측력이 없습니다.

---

### Layer 2-B: 장기 미출수 추적 (missing_weight: 20)

```python
missing_gap = latest_draw_no - last_seen
score = minmax(missing_gap)  # 오래 안 나온 번호 = 높은 점수
```

**설계 의도**: 오래 나오지 않은 번호는 "때가 됐다".

**수학적 판정**: ❌ **도박사의 오류의 가장 고전적인 형태**

> 이 오류는 1913년 몬테카를로 카지노에서 룰렛 구슬이 26번 연속 검은색에 떨어졌을 때  
> 도박꾼들이 "이제 빨간색 차례"라고 베팅해 대규모 손실을 본 사례와 동일한 논리입니다.
> 무작위 추첨에서 "미출 기간"은 예측 근거가 될 수 없습니다.

---

### Layer 2-C: 구간 분포 균형 (zone_weight: 15)

```python
# 1-9, 10-19, 20-29, 30-39, 40-45 구간 이상적 비율 대비 부족 구간 점수 상향
deficit = max(0, ideal - zone_ratios)
```

**설계 의도**: 특정 구간이 최근 적게 나왔으면 다음에 더 나온다.

**수학적 판정**: ❌ **예측 용도로는 도박사의 오류 (단, 조합 필터링에는 유효)**

> 추첨 예측에는 효과 없음.  
> 단, 생성된 조합에 구간 다양성을 강제하는 필터로 사용할 때는 의미가 있습니다.  
> (역대 당첨 조합의 약 85%가 3개 이상의 구간에서 분포)

square_weight: int = 0  # 통계 근거 없음, 비활성

## 4. 조합 생성 필터 평가 ★★★★☆

이 부분이 **실제로 의미 있는 가장 유효한 부분**입니다.

### 4-1. 수학 필터 (generation.py)

| 필터 | 근거 | 평가 |
|------|------|------|
| 총합 100~170 필터 | 역대 당첨 조합의 약 78% 해당 | ✅ 통계적으로 유효 |
| 홀짝 균형 (2:4~4:2) | 완전 치우침(0:6, 6:0)은 약 3% 미만 | ✅ 유효 |
| 고저 균형 (2:4~4:2) | 동일 근거 | ✅ 유효 |
| 3연속 이상 제외 | 역대 약 5% 미만 | ✅ 유효 |
| 구간 다양성 ≥3 | 역대 약 85% 해당 | ✅ 유효 |
| 끝자리 ≥4종 | 역대 약 94% 해당 | ✅ 유효 |
| AC값 ≥7 | 중간 이상 복잡도 필터 | ✅ 합리적 |

**이 필터들은 "역대에 자주 나온 패턴의 범위"에서만 번호를 선택하게 합니다.**  
단, 이것이 해당 조합이 당첨될 확률을 높이지는 않습니다.  
(1/8,145,060은 모든 조합에 동일)

### 4-2. 인기 패턴 회피 (popularity_risk) ★★★★★

```python
# 생일수(~31), 연속수, 끝수 몰림, 등차 패턴 감점
```

**이것이 유일하게 기댓값을 실질적으로 높이는 전략입니다.**

> **상금 기댓값 = 당첨 확률 × (상금 / 공동 당첨자 수)**
>
> 당첨 확률은 높일 수 없으나, 공동 당첨자 수는 조합 선택으로 줄일 수 있습니다.
> 인기 조합(생일수, 1-2-3-4-5-6 등)은 같은 복권을 산 사람이 수백~수천 명 있어  
> 당첨 시 상금을 1/N로 나눠야 합니다.
> 비인기 조합 선택은 당첨 확률은 동일하지만 **수령 상금을 수십~수백 배 높일 수 있습니다.**

## 5. 백테스트 결과의 의미 재해석

```python
random_avg_hits = 6.0 * candidate_count / NUMBER_MAX
# 예: 후보 20개 → 기대치 = 6 × 20/45 = 2.67개 매칭
```

시스템의 백테스트가 "무작위 대비 우수"를 보여줄 수 있습니다.  
그러나 이것이 **예측력의 증거가 되지 않는 이유**:

1. **소표본 문제**: 100~300회 백테스트는 무작위 변동성 내에 있음
2. **과적합(Overfitting)**: 역대 데이터로 만든 알고리즘이 같은 역대 데이터에 잘 맞는 것은 당연
3. **미래 예측과 무관**: 과거에 잘 맞았다고 미래에 잘 맞지 않음 (출처: 효율적 시장 가설)

**올바른 해석**: 백테스트는 "이 조합들이 역대 패턴의 범위 내에 있는가"를 보여주지,  
"다음 회차에 맞는가"를 보여주지 않습니다.

## 7. 확률 향상을 위한 실질적 권고사항

### 7-1. 구입 전략 (당장 적용 가능)

| 전략 | 효과 | 설명 |
|------|------|------|
| **비인기 조합 선택** | ★★★★★ 실질적 기댓값 향상 | 생일수, 연속수, 끝수 몰림 제외 — 이미 구현됨 |
| **더 많은 게임 구매** | ★★★★★ 유일한 확률 향상법 | 확률 = n/8,145,060 (n=게임수) |
| **QP(자동) 활용** | ★★★☆☆ 분산 효과 | 시스템 편향 없는 진정한 무작위 |

### 7-2. 알고리즘 개선 방향

#### 현재 → 권고 가중치 재설계

```python
# 현재 (예측 중심 — 근거 부족)
recent_weight   = 40  # 도박사의 오류 포함
missing_weight  = 20  # 도박사의 오류
deadzone_weight = 15  # 도박사의 오류
zone_weight     = 15  # 예측 용도로는 오류

# 권고 (필터 중심 — 근거 있음)
# 예측 점수 비중을 최소화하고, 조합 필터 강도를 높임
# 점수 기반 선택 → 통계적 범위 내에서 "다양한 비인기 조합"으로 전환
```

#### 실질 개선 항목 5가지

1. **인기도 역지수 강화**  
   인기 조합 회피 점수를 더 세분화 (지역별 인기 번호, 날짜 패턴 등)

2. **조합 커버리지 최적화**  
   5게임이 최대한 서로 다른 번호를 커버하도록 (6개 × 5게임 = 30개 중복 최소화)

3. **카이제곱 검정 도입**  
   "이 번호가 실제로 치우쳐 있는가?" 통계 검정으로 노이즈 필터링

4. **몬테카를로 시뮬레이션**  
   동일 조합 100만 회 구매 시 기대 수익 계산 → 조합별 기댓값 비교

5. **상금 풀 분석 연동**  
   인기 번호 조합의 실제 공동 당첨자 수 히스토리 분석 → 비인기 조합 선별

### 최종 결론

이 시스템은 **잘 만들어진 번호 생성기**입니다.  
수학 필터와 인기 패턴 회피를 통해 "통계적으로 흔한 범위 내에서 비인기 조합"을 생성합니다.  
이는 **당첨 시 상금을 더 온전히 받을 가능성**을 높이는 데 실제로 기여합니다.

로또는 공정한 무작위 추첨이며, 이 사실을 바꿀 알고리즘은 세상에 존재하지 않습니다.

> **유일하고 확실한 확률 향상 방법은 더 많은 게임을 구매하는 것뿐입니다.**

## 부록: 진정으로 기댓값을 높이는 최적 전략

```
1. 비인기 조합 선택 (현재 시스템의 popularity_risk 활용)
   → 공동 당첨자 감소 → 수령 상금 증가

2. 총합 100~170 범위 내 조합 선택 (현재 필터 활용)
   → 역대 패턴과 일치하는 조합군에서 선택

3. 조합 다양성 극대화 (5게임 구매 시 max_overlap: 2 이하 설정)
   → 하나의 번호에 베팅 집중 방지

4. 꾸준한 참여 (복리 효과 없음, 매 회차 독립)
   → 통계적으로 더 많이 참여할수록 당첨 가능성 증가

5. 당첨금이 큰 회차 집중 공략
   → 이월 회차(누적 당첨금 증가)에서 기댓값 일시적 상승
```
